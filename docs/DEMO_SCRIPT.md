# 다올 컨시어지 — 7단계 시연 스크립트

심사 대비 60~90초 시연 시나리오. 각 단계는 **3~5초** 내 완료 가능한 시각적 피드백 포함.

---

## 개요

| 단계 | 시간 | 내용 | 증명 포인트 |
|---|---|---|---|
| 1 | 10s | 게스트 QR 스캔 → 홈보드 + 4개국어 전환 | 진입 장벽 0 · 다국어 UX |
| 2 | 15s | AI 채팅 "베개 하나 더 주세요" → 스태프 실시간 푸시 | AI 자연어 라우팅 + WebSocket |
| 3 | 15s | 분실물 신고 → 스태프 매칭 → 회수 | 4단계 워크플로우 |
| 4 | 10s | 대여 주문 → 재고 감소 → 반납 | 원자적 재고 관리 |
| 5 | 10s | VOC URGENT 접수 → 관리자 에스컬레이션 토스트 | 심각도 기반 자동 라우팅 |
| 6 | 10s | 당직 로그 교대 → 인수인계 메모 | 운영 기록 자동화 |
| 7 | 15s | 관리자 리포트 — 일일 통계 + SLA + 히트맵 | 데이터 기반 운영 인사이트 |

**총 ~90초**

---

## 단계별 세부 시나리오

### STEP 1 — 게스트 진입 (10초)
**준비**: 모바일 기기 · 객실 302호 QR 인쇄물

1. 카메라로 QR 스캔 → 앱이 자동 진입
2. 홈보드 상단 언어 셀렉트 → 🇰🇷 ko → 🇺🇸 en → 🇯🇵 ja → 🇨🇳 zh
3. **설명 멘트**: "로그인 없이 객실 QR 한 번으로 72시간 토큰 발급, 4개 국어 즉시 전환."

### STEP 2 — AI 자연어 요청 (15초)
**화면**: 게스트 채팅 + 스태프 대시보드 (분할)

1. 채팅창에 입력: **"베개 하나 더 주세요"**
2. AI 응답: "확인 중... 어메니티를 요청했습니다."
3. 동시에 스태프 대시보드 **🛎️ 어메니티** 카드가 슬라이드-인 (WebSocket)
4. **설명 멘트**: "Claude API 가 자연어 의도를 파싱 → 어메니티 도메인 자동 분류 → HK 부서 실시간 푸시. 룰 기반 폴백 내장."

### STEP 3 — 분실물 신고/회수 (15초)
**화면**: 게스트 LostFoundView + 관리자 AdminLostFoundView

1. 게스트: 카테고리 👛 Wallet 선택, 물품명 "검은색 반지갑", 위치 "702호 침대 옆" 입력 → 신고
2. 관리자 테이블에 신규 행 추가 (REPORTED)
3. 관리자: "습득처리" 클릭 → FOUND 상태 변경
4. 다시 "반환완료" 클릭 → RETURNED
5. **설명 멘트**: "게스트 신고 → 스태프 발견 기록 → 매칭 → 회수. 4단계 상태 기계."

### STEP 4 — 대여 주문 (10초)
**화면**: 게스트 RentalView + 관리자 AdminRentalView

1. 게스트: ☂️ Umbrella 카드 → qty=1 → Request Rental
2. 카탈로그 재고 뱃지 즉시 갱신 (5/5 → 4/5)
3. 관리자 주문 탭에 REQUESTED 행 추가 → "대여 출고" → LOANED
4. 반납 버튼 → 재고 복구 (4/5 → 5/5)
5. **설명 멘트**: "재고 감소/복구는 DB `STOCK_AVAILABLE = STOCK_AVAILABLE + delta` 원자적 UPDATE 로 race 방지."

### STEP 5 — VOC URGENT 에스컬레이션 (10초)
**화면**: 게스트 VocView + 관리자 StaffDashboardView

1. 게스트: 카테고리 🏢 Facility, **심각도 URGENT (빨간 pill)**, 제목 "샤워기 물 안 나옴", 내용 입력 → 제출
2. 관리자 대시보드 **우상단에 빨간 🚨 SLA 초과/에스컬레이션 토스트** 페이드인
3. 동시에 AdminVocView 행 추가 (빨간 border-left)
4. **설명 멘트**: "URGENT 심각도는 `/topic/ccs/esc/{propCd}` 전용 토픽으로 관리자 토스트 즉시 푸시."

### STEP 6 — 당직 로그 교대 (10초)
**화면**: 스태프 DutyLogView

1. 스태프: ▶ Night Shift 버튼 클릭 → 로그 생성
2. Summary 필드에 "22:00 접수 3건 완료. 9층 누수 이슈 → ENG 인계."
3. Handover To 입력 "최영수(주간)" → Handover 버튼
4. 관리자 AdminDutyView 에서 Today 카드 갱신 확인
5. **설명 멘트**: "근무 시작 · 인수인계 · 종료 3 단계. PMS 야간감사 상태 배지 연동 (read-only)."

### STEP 7 — 관리자 리포트 (15초)
**화면**: AdminReportsView

1. 기본 탭 **일일 통계**: 최근 7일 요청/완료/평균시간, SLA MET/MISSED 뱃지
2. **SLA 준수율** 탭 전환: 부서×소스타입별 completionRate 그라데이션 바
3. **히트맵** 탭: 요일(일~토) × 시간(0~23) 24 셀 네이비 그라데이션
4. CSV 내보내기 버튼 클릭 → 파일 다운로드
5. **설명 멘트**: "5 본연 기능 데이터 포함, 외부 차트 라이브러리 없이 CSS 로 구현. SaaS 납품 전제 조건 충족."

## 시연 팁

### 기기 셋업
- **왼쪽 화면**: 모바일 (게스트 앱, iPhone 권장) — QR 스캔 준비
- **오른쪽 화면**: 노트북 (스태프/관리자 브라우저) — 두 탭 병렬 (대시보드 + 현재 시연 도메인)
- 볼륨: ZERO (WebSocket 푸시 비주얼 피드백만으로 충분)

### 시연 전 체크
1. `ANTHROPIC_API_KEY` 설정 (AI 폴백 아닌 실제 응답)
2. 테스트 데이터: 어메니티 5 품목 + 대여 카탈로그 5 품목 사전 투입
3. 스태프 로그인 (`/staff/login`): 시드 계정 `fr1 / test1234` (프론트), `hk1 / test1234` (HK), `eng1`, `fb1`. 관리자는 PMS_USER_MTR `USER_TP=00001/00002/00003` 계정으로 로그인 → `/staff/context` 호텔 선택 단계 진입
4. 객실 03010 의 체크인 일자가 오늘인지 확인 (QR 토큰 유효)

### 폴백 시나리오 (네트워크 실패 등)
- 녹화된 demo.mp4 재생 (landing/index.html 의 video 섹션)
- 로컬에서만 작동하는 `concierge.tenant.prop-cd=MOCK` 환경

### 심사 질의 예상 응답
- **"PMS 연동은?"** — PmsSyncAdapter stub 활성화 상태. VPN 접속 시 실제 테이블 INSERT 로 교체.
- **"보안은?"** — 게스트 JWT(72h) + 스태프 JWT(`USER_TP` 클레임 + 호텔 컨텍스트) 2 계층. 어드민 메뉴는 `MenuAccess.assertCanAccess` 백엔드 가드 + 라우터/메뉴 가드 이중 방어 (`X-Admin-Token` 단일 비밀번호 방식 폐기, 2026-04-22).
- **"확장성?"** — 멀티테넌시(propCd/cmpxCd) + 기능 플래그 + WebSocket 토픽 역할별 계층. 프로퍼티 추가는 INV 행 삽입만.
